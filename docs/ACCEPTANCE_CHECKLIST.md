# Modelling and Tools Study 验收清单

本清单用于本地发布前回归。最近一次完整验收日期：2026-06-29。

## 本地启动

```bash
npm ci
npm run dev
```

开发服务器默认从根路径提供应用，例如 `http://localhost:5173/`。学习记录保存在当前浏览器的 `f11mt-study-state-v1` localStorage key 中。

生产构建和本地预览：

```bash
npm run deploy:build
npm run preview
```

生产构建使用 GitHub Pages 仓库路径 `/modling/`，预览地址通常为 `http://localhost:4173/modling/`。

## 实验室验收

### 矩阵乘法

- [ ] “下一格”依次切换当前行、当前列和点积过程。
- [ ] 正确矩阵 `[[4,4],[10,8]]` 得到完整正确提示。
- [ ] 单格错误时明确指出错误的行列位置。

### 动态系统与递推关系

- [ ] Logistic map 默认使用 `r=2.8`、`x₀=0.2`、`steps=20`。
- [ ] 修改参数并点击“模拟”后，折线图与 `k,x_k` 数值表同步更新。
- [ ] `r` 不在 0 到 4、`x₀` 不在 0 到 1 或步数无效时显示明确错误。
- [ ] 行为提示能区分收敛、周期震荡和复杂行为的直觉。
- [ ] 二阶递推从 `y₀=[1,0]^T` 开始，每次“下一步”只推进一项。
- [ ] 连续推进后显示 `x₂=2,x₃=2,x₄=0,x₅=-4,x₆=-8`。
- [ ] 页面显示 `A=[[2,-2],[1,0]]`、`y_{k+1}=Ay_k` 和当前矩阵乘法过程。
- [ ] “重新开始”恢复序列 `[0,1]`；手机宽度下页面没有横向溢出。

### 图论

- [ ] 默认 K4 显示 4 个顶点、6 条有效边。
- [ ] 关闭一条边后，边数与对应按钮状态同步变化。
- [ ] 邻接矩阵 A、度矩阵 D、Laplacian `L=D-A` 同步更新。
- [ ] 恢复 K4 后重新显示 6 条边和默认矩阵。
- [ ] `v1` 到 `v3`、长度 4 的 walks 显示 20。

### 概率密度

- [ ] 默认 `P(0≤X≤0.5)` 显示 `1/8 = 0.125`。

- [ ] `E[X]` 显示 1。

## 已知限制

- 复杂自由文本证明仍不自动评分；当前训练器只检查步骤顺序和关键空。
- 50 分结构模拟卷可用，但不是完整的严格两小时考试体验。
- 学习数据没有账号或云同步；跨设备需要手动导出和导入 JSON。
- 数学、图表和图可视化依赖仍集中在主 bundle，首次加载体积有进一步拆分空间。
- 当前主 bundle 较大；数学、图表和图可视化库尚未按路由拆包。
- `npm audit` 仍报告一个与 Windows 本地 Vite 开发服务器相关的 low 级 esbuild 问题；生产静态站点不运行开发服务器。
- 今日复习依赖设备日期，跨时区或手动修改系统时间会影响到期判断。
