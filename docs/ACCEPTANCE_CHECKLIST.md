# Modelling and Tools Study 验收清单

本清单用于本地发布前回归。最近一次完整验收日期：2026-06-28。

## 本地启动

```bash
npm ci
npm run dev
```

开发服务器默认从根路径提供应用，例如 `http://localhost:5173/`。学习记录保存在当前浏览器的 `f11mt-study-state-v1` localStorage key 中。

生产构建和本地预览：

```bash
npm run deploy:build
npm run preview
```

生产构建使用 GitHub Pages 仓库路径 `/modling/`，预览地址通常为 `http://localhost:4173/modling/`。

## GitHub Pages 准备

- Vite 生产构建 base 为 `/modling/`，本地开发 base 保持 `/`。
- Hash router 不依赖服务器端路由回退，适合静态托管。
- 当前仓库尚未启用 GitHub Pages，因此没有启用自动部署 workflow，避免 `main` 推送后产生预期失败。
- 启用方法：在 GitHub 仓库 `Settings > Pages` 中将 Source 设为 `GitHub Actions`，再加入官方 Pages artifact/deploy workflow。

## 已知限制

- 动态系统与递推关系仍是预告模块。
- 复杂自由文本证明题仍采用步骤填空和选项的 beta 判分。
- 50 分结构模拟卷可用，但不是完整的严格两小时考试体验。
- 学习数据只保存在当前浏览器，没有账号同步、导入或导出。
- 当前主 bundle 较大；数学、图表和图可视化库尚未按路由拆包。
- `npm audit` 仍报告一个与 Windows 本地 Vite 开发服务器相关的 low 级 esbuild 问题；生产静态站点不运行开发服务器。
- 今日复习依赖设备日期，跨时区或手动修改系统时间会影响到期判断。
